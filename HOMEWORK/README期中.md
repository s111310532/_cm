求解經典單擺運動(用gemini)
1. 降階思想
電腦通常不擅長直接處理高階微分方程（如二階 y''）。因此，我們將二階方程轉化為一對聯立的一階微分方程組。這就像是把「加速度」拆解為「速度的變化」，再把「速度」拆解為「位置的變化」。
2. 為什麼選擇 RK4？
RK4 ：這是一種「預測-修正」機制。它不只看當前點的斜率，還計算了區間中點與終點的四個斜率樣本 (k_1到k_4) 並加權平均。這使得它的精確度非常高。
3. 處理非線性
   與你之前詢問的「常係數齊次方程」不同，單擺方程包含 sin(theta)，這是非線性的。傳統的特徵方程式法（求根法）在此失效，必須依靠這種迭代數值解法。
4.向量化運算
在程式中，我使用 state = np.array([theta, omega])。這樣做的好處是可以一次性更新所有變數，程式碼更簡潔且易於擴展到更多維度的系統（例如雙擺或行星軌道）。
